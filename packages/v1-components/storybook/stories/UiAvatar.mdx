# UiAvatar

Аватар пользователя, используемый для визуальной идентификации

## Описание
В системе используются несколько размеров аватарок в зависимости от их места расположения. Также, аватарки могут быть с пометками VIP/BAD
в карточке клиента или списках.

### Big — 52px
* карточка клиента
* профиль клиента в чате
* карточка менеджера (в будущем)

<ToReact is={ExampleSize} size={'lg'}/>

### Small — 36px
* список клиентов
* список диалогов в чате
* переписка в чате
* карточка участия в программе лояльности (ПЛ)
* таймлайн в участии ПЛ

<ToReact is={ExampleSize}/>

### Small — 24px
* история в карточке задачи

**Данный размер используется в системе без иконок vip/bad и без статуса**

<ToReact is={ExampleSize} size={'xs'} />

### Статусы
В зависимости от размера аватарки устанавливаются соответствующие отступы. Диаметр статусных иконок - **8px**.
Для аватарок большого размера (`LG`) отступ от краёв составляет **4px**.

<ToReact is={ExampleStatus} size={'lg'}/>

Для маленьких иконок - **2px**.

<ToReact is={ExampleStatus} />

### Состояние с пустой аватаркой
При отсутствии в компоненте ссылки на изображение и инициалов используется иконка с вопросительным знаком.

<ToReact is={ExampleEmptyAvatar} />

### Комбинации

Использование нескольких аватарок рядом приводит к добавлению белой обводки **2px**. «Нахлест» составляет **4px** у medium (`SM`),
**8px** у big (`LG`) и **2px** у small (`XS`).

<ToReact is={ExampleList} count-avatars={3} />

Если пользователей больше 5, то остальные аватарки схлопываются и показываются числом. В некоторых местах системы это правило используется
ситуативно. В случае необходимости можно схлопывать аватарки после 3-х.

<ToReact is={ExampleList} count-avatars={6} />

## Оптимизация

Компонент аватара может автоматически уменьшить размер загружаемого изображения за счет формирования URL специального
вида, который строится с помощью утилиты `@retailcrm/image-preview`.

Оптимизация включена по-умолчанию, свойство `optimize` компонента может принимать значения:

* `true` — использовать оптимизацию по-умолчанию
* `false` — отключить оптимизацию
* `{ resize?: Dimensions, crop?: Dimensions }` — параметры изменения размера и обрезки; `Dimensions` — строки вида
 `36x36`, `72x-`, т.е. `NxM`, где N — некоторое число (x-ось), M — некоторое число (y-ось) или прочерк (дефис).
