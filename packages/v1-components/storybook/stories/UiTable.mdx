# UiTable

`UiTable` - декларативная slot-based таблица для remote-части `embed-ui`.
Компонент собирается из `UiTable` и дочерних `UiTableColumn`, а сложные сценарии добираются через слоты `expand`, `group-head` и footer-slots.

## Когда использовать

* Нужна таблица с ручной композицией колонок и ячеек.
* Нужны grouping, expandable rows или кастомный footer без жёсткого data-driven API.
* Нужно сохранить разделение `host`-примитивов и `remote`-orchestration, как у `UiSelect`.

## Контент ячеек

`UiTable` рассчитана на типовые сценарии контента в `header` и `body`, а в историях и прикладном коде можно сразу опираться на готовые `v1-components`.

Для body-ячеек подходят такие комбинации:

* `Text` или `Text + description` - обычный текст, ссылка через <a className="ui-table-docs__component-link" href="?path=/docs/components-uilink--docs"><code>UiLink</code></a>, а вторую строку удобно собирать простым `div/span`.
* `Status` - статусные плашки через <a className="ui-table-docs__component-link" href="?path=/docs/components-uitag--docs"><code>UiTag</code></a>.
* `Avatar + Text` - типовой сценарий клиента, менеджера или пользователя через <a className="ui-table-docs__component-link" href="?path=/docs/components-uiavatar--docs"><code>UiAvatar</code></a> и <a className="ui-table-docs__component-link" href="?path=/docs/components-uilink--docs"><code>UiLink</code></a> или обычный текст.
* `Icon 20x20` - декоративные и action-иконки, если нужен только маркер состояния или компактное действие.
* `Button` - обычные кнопки дизайн-системы, когда внутри ячейки нужно явное действие, например через <a className="ui-table-docs__component-link" href="?path=/docs/components-uibutton--docs"><code>UiButton</code></a>.
* `Checkbox` - выделение строки или массовые действия через <a className="ui-table-docs__component-link" href="?path=/docs/components-uicheckbox--docs"><code>UiCheckbox</code></a>.
* `Image` - превью товара или сущности, если в колонке нужен визуальный контент.

Для header обычно хватает таких паттернов:

* `Text` - обычный заголовок колонки.
* `Text + Icon` - сортировка или вспомогательное действие; для сортировки лучше использовать <a className="ui-table-docs__component-link" href="?path=/docs/components-uitable--docs"><code>UiTableSorter</code></a>.
* `Checkbox + Icon` - service column с массовым выделением и дополнительным действием через <a className="ui-table-docs__component-link" href="?path=/docs/components-uicheckbox--docs"><code>UiCheckbox</code></a>.
* `Icon` или `Icon + Text + Icon` - узкие служебные колонки и более сложные action-header сценарии.

### Таблица с группировкой

<Story of={UiTableStories.GroupedRows} />

```vue
<template>
  <UiTable
    bordered
    :rows="rows"
    row-key="id"
    :group-by="groupByStatus"
  >
    <UiTableColumn label="Название">
      <template #cell="{ row }">
        {{ row.title }}
      </template>
    </UiTableColumn>

    <UiTableColumn label="Статус" width="160">
      <template #cell="{ row }">
        {{ row.status }}
      </template>
    </UiTableColumn>

    <template #group-head="{ group }">
      <span>{{ group.data.title }}</span>
    </template>
  </UiTable>
</template>

<script lang="ts" remote setup>
import type { Group } from '@/common/components/table'
import { UiTable, UiTableColumn } from '@/remote/components/table'

const rows = [
  { id: 101, title: 'Весенняя рассылка', status: 'Запланирована' },
  { id: 102, title: 'Скидка на аксессуары', status: 'Отправлена' },
]

const groupByStatus = (items: readonly typeof rows): Group<(typeof rows)[number], { title: string }>[] => {
  return ['Запланирована', 'Отправлена']
    .map((status) => ({
      key: status,
      data: { title: status },
      rows: items.filter((row) => row.status === status),
    }))
    .filter((group) => group.rows.length > 0)
}
</script>
```

## Важные свойства

* `headless` - скрывает стандартный header.
* `bordered` - включает внешний контур таблицы.
* `fixed` - переводит таблицу в `table-layout: fixed`.
* `rows` и `rowKey` - основной источник данных и стабильных ключей строк.

## Важные слоты

* `default` - список колонок `UiTableColumn`.
* `expand` - раскрытое содержимое строки.
* `group-head` - заголовок группы.
* `footer`, `footer-summary`, `footer-page-size`, `footer-export`, `footer-pagination` - footer API.

## CSS tokens

Стили `UiTable` можно настраивать через CSS custom properties на корневом элементе таблицы.

* `--ui-v1-table-border-width` - толщина внешней рамки и базовая величина для border-aware расчётов внутренних отступов.
* `--ui-v1-table-rounding`, `--ui-v1-table-rounding-start-start`, `--ui-v1-table-rounding-start-end`, `--ui-v1-table-rounding-end-start`, `--ui-v1-table-rounding-end-end` - общее и покомпонентное скругление таблицы.
* `--ui-v1-table-cell-padding-x`, `--ui-v1-table-cell-padding-y` - базовые горизонтальные и вертикальные padding всех ячеек.
* `--ui-v1-table-cell-padding-block-start`, `--ui-v1-table-cell-padding-inline-end`, `--ui-v1-table-cell-padding-block-end`, `--ui-v1-table-cell-padding-inline-start` - точечные переопределения padding ячейки по logical-сторонам.
* `--ui-v1-table-padding-start`, `--ui-v1-table-padding-end` - внешние отступы первой и последней ячейки в строке; в `bordered`-варианте таблица сама вычитает из них ширину рамки.
* `--ui-v1-table-head-cell-padding-block-start`, `--ui-v1-table-head-cell-padding-block-end` - отступы header-ячейки по block-оси.
* `--ui-v1-table-body-cell-padding-block-start`, `--ui-v1-table-body-cell-padding-block-end` - отступы body-ячейки по block-оси.
* `--ui-v1-table-group-head-padding-block-start`, `--ui-v1-table-group-head-padding-block-end` - отступы строк-заголовков групп по block-оси.
* `--ui-v1-table-head-background` - фон шапки таблицы.
* `--ui-v1-table-group-head-background` - фон заголовков групп.
